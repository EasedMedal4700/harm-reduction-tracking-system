# Theme Rules for Flutter App

## Overview
This document outlines the rules for using the theme system in the Flutter app. The theme is designed to be centralized, consistent, and compliant with the project's architecture ruleset. All theme-related values must come from the provided constants and extensions to ensure maintainability and scalability.

## Key Principles
- **No Hardcoding**: Never hardcode colors, fonts, sizes, or spacing. Always use the theme extensions and constants.
- **Centralized Theme**: All theme values are defined in `lib/constants/theme/` and accessed via `BuildContext` extensions.
- **Consistency**: Use the same theme values across all widgets to maintain a uniform look.
- **Responsiveness**: Theme values account for different screen sizes and accessibility needs.

## How to Use Theme in Code

### Colors
- Use `context.colors` for all color needs.
- Examples:
  - `context.colors.primary` for primary colors.
  - `context.colors.background` for backgrounds.
  - `context.colors.error` for error states.
- Do not use `Colors.red` or hex codes directly.

### Typography
- Use `context.text` for all text styles.
- Examples:
  - `context.text.headlineLarge` for large headings.
  - `context.text.bodyMedium` for body text.
- Do not use `TextStyle()` with hardcoded values.

### Spacing
- Use `context.spacing` for padding, margins, etc.
- Examples:
  - `context.spacing.sm` for small spacing.
  - `context.spacing.lg` for large spacing.
- Do not use `SizedBox(height: 16)` or similar hardcoded values.

### Sizes and Layout
- Use constants from `app_sizes.dart`, `app_layout.dart`, etc., accessed via context if available.
- Examples:
  - `AppLayout.mainAxisAlignmentCenter` for alignment.
  - `context.sizes.iconSize` for icon sizes.

### Animations, Borders, Shadows, etc.
- Use the respective constants from `app_animations.dart`, `app_borders.dart`, `app_shadows.dart`, etc.
- Access via context extensions where possible.

## Detailed Theme Constants

### Padding (Spacing)
All spacing values are in `app_spacing.dart`. Use `context.spacing` for access.

- **xs**: 4.0 (extra small)
- **sm**: 8.0 (small)
- **md**: 12.0 (medium)
- **lg**: 16.0 (large)
- **xl**: 24.0 (extra large)
- **xl2**: 32.0 (2x extra large)
- **xl3**: 48.0 (3x extra large)
- **space32**: 32.0
- **space48**: 48.0
- **cardSpacing**: 16.0
- **homePagePadding**: 16.0
- **quickActionSpacing**: 16.0
- **cardPaddingSmall**: 12.0
- **cardPadding**: 16.0
- **cardMargin**: 12.0

Compact mode (reduced by 25%):
- **xsCompact**: 3.0
- **smCompact**: 6.0
- **mdCompact**: 9.0
- **lgCompact**: 12.0
- **xlCompact**: 18.0
- **xl2Compact**: 24.0
- **xl3Compact**: 36.0
- **cardPaddingCompact**: 12.0
- **cardMarginCompact**: 8.0

### Radius
All radius values are in `app_radius.dart` and `app_shapes.dart`. Use `AppRadius` for access.

- **xs**: 4.0 (extra small)
- **sm**: 8.0 (small)
- **md**: 12.0 (medium)
- **lg**: 16.0 (large)
- **xl**: 20.0 (extra large)
- **xl2**: 24.0 (2x extra large)
- **full**: 999.0 (full radius for circles)

Additional:
- **buttonRadius**: 12.0
- **quickActionRadius**: 16.0

### Typography
All typography styles are in `app_typography.dart`. Use `context.text` for access. Styles are responsive and adapt to user font size settings.

Headings:
- **heading1** / **displayLarge**: Large heading (scaled from 28.0)
- **heading2** / **displayMedium** / **headlineLarge**: Medium heading (scaled from 24.0)
- **heading3** / **displaySmall** / **headlineMedium**: Small heading (scaled from 20.0)
- **heading4** / **titleLarge**: Subheading (scaled from 18.0)
- **titleSmall**: Small title (scaled from 12.0)

Body Text:
- **bodyLarge**: Large body (scaled from 16.0)
- **body** / **bodyMedium**: Standard body (scaled from 14.0)
- **bodyBold** / **titleMedium**: Bold body (scaled from 14.0)
- **bodySmall**: Small body (scaled from 12.0)

Utility Text:
- **caption** / **labelSmall**: Caption (scaled from 11.0)
- **captionBold** / **labelMedium**: Bold caption (scaled from 11.0)
- **overline**: Overline (scaled from 10.0)
- **label**: Label (scaled from 11.0)

Buttons:
- **button** / **labelLarge**: Button text (scaled from 14.0)
- **buttonSmall**: Small button text (scaled from 12.0)

Font Constants:
- **fontXSmall**: 12.0
- **fontSmall**: 14.0
- **fontMedium**: 16.0
- **fontLarge**: 18.0
- **fontXLarge**: 20.0
- **font2XLarge**: 24.0
- **font3XLarge**: 32.0
- **fontBold**: FontWeight.w700
- **fontSemiBold**: FontWeight.w600
- **fontMediumWeight**: FontWeight.w500
- **lineHeightTight**: 1.2
- **fontFamilyMonospace**: 'monospace'

## Migration and Compliance
- In migration headers, mark "Theme: COMPLETE" if the file uses theme extensions properly.
- Mark "Theme: LEGACY" if it has hardcoded valuesâ€”refactor immediately.
- All new code must be "Theme: COMPLETE".

## Examples
```dart
// Good: Using theme
Container(
  color: context.colors.surface,
  padding: EdgeInsets.all(context.spacing.md),
  child: Text(
    'Hello',
    style: context.text.bodyLarge,
  ),
)

// Bad: Hardcoding
Container(
  color: Colors.white,
  padding: EdgeInsets.all(16),
  child: Text(
    'Hello',
    style: TextStyle(fontSize: 16),
  ),
)
```

## Enforcement
- Violations will be flagged in code reviews.
- Use these rules when prompting AI tools like ChatGPT for theme-related changes.
- Ensure all theme files in `lib/constants/theme/` are up-to-date and comprehensive.
